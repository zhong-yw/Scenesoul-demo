> **文档标识：** 13-流程/01-用户交互全流程  
> **对应代码：** [main.py](main.py) `ScenesoulLoop.handle_user_input()` + `handle_user_timeout()`  
> **版本：** v0.5（Message 系统重构）

## 一、用户交互的四个阶段

```
① 用户到达 ──▶ ② 对话 ──▶ ③ 沉默 ──▶ ④ 离开
                │                   │
                └── 循环 ────────────┘
```

### 阶段一：用户到达

用户发送第一条消息时触发。此时大脑可能在独白，界说需要"打断"大脑的独白，将用户自然地融入场景。

### 阶段二：对话

用户持续发送消息，界说将消息注入场景，大脑回应。循环往复。

### 阶段三：沉默

用户停止发送消息，但仍可能回来。系统进入"等待"状态——等待超时（600秒）或新消息。

### 阶段四：离开

用户超时未发言，界说构造用户离开场景旁白，系统恢复互相唤醒循环。

## 二、详细流程：用户发送消息

### CLI 模式

```
用户按 Enter 提交
    │
    ├──▶ / 开头 → renderer.execute_command(command)
    │
    └──▶ 正常文本 → ScenesoulLoop.handle_user_input(text)
            │
            ├──① 更新 last_user_time = now
            ├──② user_present = True
            │
            ├──③ 界说构建到达/注入旁白
            │    ├── narrator_messages 为空 → narrator.handle_user_arrival()
            │    └── narrator_messages 非空 → narrator.handle_user_message()
            │
            ├──④ 界说旁白 → 追加到两个消息列表
            │    ├── narrator_messages.append(assistant, narration)
            │    ├── 拼接 [当前状态] 头部
            │    └── brain_messages.append(user, 头部+旁白)
            │
            ├──⑤ brain.respond(brain_messages, user_input)
            │    → 返回回应文本
            │    → 追加到 brain_messages (assistant)
            │
            └──⑥ narrator.observe(brain.last_thought)
                 → 界说观测大脑回应
                 → 可能触发 scene_change + tool_call
```

### Web 模式

```
前端 POST /api/send {"message": "你好"}
    │
    ├──▶ brain_messages 追加 user_input
    ├──▶ narrator.handle_user_arrival() 或 handle_user_message()
    ├──▶ 拼接 [当前状态] → brain_messages
    ├──▶ brain.respond() → 追加 brain_messages
    ├──▶ narrator.observe() → 处理场景变化
    └──▶ 返回 {"status": "ok"}
          → 前端 refreshConversation()
```

## 三、详细流程：用户沉默与超时

```
用户无发言
    │
    ├──▶ 0-600 秒：normal（正常等待）
    │     大脑继续独白（THINK_INTERVAL=10 秒）
    │     界说继续观测
    │
    └──▶ > 600 秒 → 超时
            │
            ├──▶ user_present = False
            ├──▶ renderer.print_system("用户安静离开了……")
            ├──▶ narrator.handle_user_leave() → 离开旁白
            ├──▶ 离开旁白 → 两个消息列表
            ├──▶ 大脑继续思考 + 界说观测
            └──▶ 恢复互相唤醒循环
```

### 超时后的状态

```
user_present = False
last_think_time = now  // 重新计时
```

## 四、消息列表变化

### 用户消息处理时的消息列表追加

```
处理前:
  brain_messages: [... user: 世界消息, assistant: 独白]
  narrator_messages: [... user: 大脑独白, assistant: 界说输出, tool_call, tool_result]

用户输入: "你好"

处理后:
  narrator_messages追加:
    assistant: 厨房门口传来一个声音:"你好"你回过头……

  brain_messages追加:
    user: [当前状态] 时间:… 场景:厨房 驱动力:…
          厨房门口传来一个声音:"你好"你回过头……

  brain_messages再追加:
    assistant: 嗯？——我在厨房呢。你来得正好。

  narrator_messages再追加:
    user: 嗯？——我在厨房呢。你来得正好。

  narrator_messages再追加:
    assistant: [推进] 你转身看向门口……
```

## 五、边缘情况

### 5.1 用户超时瞬间又发消息

```
if user_present and now - last_user_time > USER_TIMEOUT:
    # 超时处理
    user_present = False
    handle_user_timeout()

# 紧接着用户发消息
# → handle_user_input()
# → user_present 为 False（刚刚重置）
# → 走 arrival 流程（正确）
```

用户会在终端看到"他离开了"紧跟着"门开了他走进来"，这是设计接受的行为。

### 5.2 用户连续快速发送多条消息

CLI 模式 50ms 轮询，InputLine 的 buffer 可以累积，Enter 提交时将全部内容一次性提交，对消息处理没有影响。

### 5.3 多人同时使用 Web UI

当前 Web UI 只有一个全局的 ScenesoulLoop 实例，多人同时使用时消息会串扰。这是设计限制——v0.5 假设单用户使用。
