## 📋 今日推进纪要（5/3 晚 → 5/4 凌晨）

### 重大架构变更

1. **多轮消息上下文 + 20K token 裁剪**
   - 大脑和界说各自维护独立 messages 列表
   - 每轮追加 user+assistant 对，超出 20K 时丢弃最旧轮次
   - 保留 system prompt + 至少最近 2 轮

2. **界说输出从 JSON 改为纯文本标记**
   - 移除 tool call 和 JSON 解析，改为 `[场景:名称]` / `[推进]` / 纯文本
   - 消除 MiniMax API 的 JSON 截断/解析错误问题

3. **CLI 界面大重构**
   - 底部固定输入行（Claude Code 风格），单线程 50ms 轮询 msvcrt.kbhit
   - 文字前缀区分：[独白]/[旁白]/[你]/[系统]
   - write_output 精确重绘，消除多余空白行

4. **大脑/界说分工明确**
   - 大脑 → 第一人称"我"，内心感受，不做场景描写
   - 界说 → 第二人称"你"，场景叙述和行动描写
   - 附带正反示例防止混淆

5. **时间流逝机制**
   - 界说观测时跟踪 quiet_rounds，连续 3+ 轮安静则主动推进时间
   - 大脑提示词加入"时间一直在流动"
   - 驱动力每轮自动变化：fatigue +3, hunger +1, curiosity -1

6. **用户消息处理完整流程**
   - establish 到达 → 界说描述到达场景 → 大脑回应
   - inject 后续消息 → 界说注入场景 → 大脑回应
   - 超时 10 分钟 → 界说描述离开 → 恢复无用户循环

### 当前运行效果
```
[独白] 嗯…这么晚了啊，我翻书翻得都把时间忘了。窗外还有鸟叫声，
       可听着一点都不吵闹，反而让心里更安静了。
[旁白] 月光里，鸟鸣声逗留在书页上。
[独白] 嗯……这段话还在心里打转呢，像羽毛似的，飘啊停的。
[旁白] 枕边书页，月光轻颤。
```
问题：大脑容易在一个情感状态循环（如上例连续 5 轮都说"舍不得睡"），
需要界说在安静时主动推进时间。
