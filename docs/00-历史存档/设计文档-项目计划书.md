## 核心总结

白界是一个双层 Agent 协作的虚拟生命框架：

| 角色 | 类比 | 核心能力 |
|------|------|----------|
| 🧠 大脑 Agent | 意识/灵魂 | 24/7 持续思考、第一人称"我"的内心独白、情绪驱动、记忆 |
| 🎭 界说 Narrator | 世界管理者 | 创造场景、第二人称"你"的旁白叙述、观测推演、时间推进 |
| 👤 用户 | 访客 | 直接对话，界说将其融入场景，大脑自然回应 |

**核心循环（互相唤醒）：**
```
启动：界说构造初始场景（卧室醒来）
无用户时：
  大脑内心独白（我第一人称感受）
  → 界说观测/推演（推进场景/流逝时间）
  → 大脑感知变化后再次独白
  → 循环……
用户到达：
  → 界说描述用户到达场景
  → 大脑回应用户
  → 用户继续发言 → 界说注入场景 → 大脑回应
  → 超时 10 分钟 → 界说描述离开 → 恢复无用户循环
```

**输出格式：**
```
[独白] 大脑的内心——第一人称"我"，口语化
[旁白] 界说的叙述——第二人称"你"，场景描写
[你] 用户消息回显
```

## 架构要点

### 独立消息上下文
- 大脑和界说各自维护独立的 messages 列表
- 互不交叉，通过函数参数传递文本信息
- 每轮追加 user + assistant，20K token 预算裁剪
- 裁剪策略：保留 system + 最近至少 2 轮，从最旧对开始丢弃

### 界说三模式
1. **推演（observe）** — 无用户时观测大脑，标记化输出
2. **接入（arrive/inject）** — 用户到达/发言时注入场景
3. **离开（leave）** — 用户超时后描述离开

### 标记化输出协议
| 输出格式 | 含义 | 示例 |
|---------|------|------|
| `[场景:名称]` + 旁白 | 切换场景 | `[场景:厨房] 你走进厨房。` |
| `[推进]` + 旁白 | 在当前场景推进 | `[推进] 你伸手拿起水壶。` |
| 纯文本旁白 | 安静场景/氛围 | `阳光洒在被子上，暖融融的。` |

### 时间感知与驱动力
- 大脑每轮思考前 `tick_drives()`：疲劳 +3/轮，饥饿 +1/轮，好奇 -1/轮
- 界说观测时感知 quiet_rounds，连续 3+ 轮安静则主动推进时间
