## 一、项目定位

**白界 · Scenesoul** 是一个"虚拟生命"框架。核心是两个 Agent 协作：
**大脑（Brain）** 作为意识体，**界说（Narrator）** 作为世界构造者。

### 核心理念

| 理念 | 说明 |
|------|------|
| 空白世界 | 大脑没有物理感知，活在纯意识空间中 |
| 记忆即灵魂 | 过去的经验积累构成"自我" |
| 界说为桥 | 大脑与外界唯一的接口——创造 + 叙述 + 观测 |
| 直接返回 | 大脑输出原汁原味，不经中间层加工 |
| Profile 即配置 | 所有人格/世界/场景通过 Markdown 文件定义 |

### 角色定位

| 角色 | 人称 | 核心能力 |
|------|------|----------|
| 🧠 大脑 Agent | 第一人称"我" | 24/7 持续思考、内心独白、情绪驱动、记忆积累 |
| 🎭 界说 Narrator | 第二人称"你" | 创造场景、旁白叙述、观测推演、维护 scene.md |
| 👤 用户 | 访客 | 直接对话，界说将其融入场景 |

## 二、模块总览

### 系统层
- **LLM Client** — 封装 OpenAI 兼容 API 调用，支持普通 chat 和 tool_call 两种模式
- **上下文构造器** — Brain/Narrator 各自独立的 context builder，接收外部消息列表构建 LLM 上下文，20K token 裁剪

### 大脑层
- **BrainAgent** — 意识核心：内心独白、用户响应；不持有消息历史，驱动力由 `ScenesoulLoop` 管理
- **BrainMemory** — L1 工作记忆 + L2 每日过程日志
- **BrainContextBuilder** — 接收外部 brain_messages 列表，动态重写固定 system 的场景列表和当前状态

### 界说层
- **NarratorAgent** — 世界管理：观测（observe）/ 用户到达（handle_user_arrival）/ 用户发言（handle_user_message）/ 用户离开（handle_user_leave）四模式，支持 `update_scene` 和 `update_drives` tool_call
- **NarratorMemory** — S1 场景状态 + S3 事件历史
- **NarratorContextBuilder** — 接收外部 narrator_messages 列表，观测任务已整合到固定 system 中

### Profile 配置层
- **ProfileLoader** — 从 `profiles/` 下的 .md 文件加载预设，动态组装 brain 和 narrator 的 system prompt（含固定指令）
- 6 份文档：soul.md / memory.md / brain.md / narrator.md / world.md / scene.md

### 世界层
- **WorldBuilder** — 世界设定调度器，profile 模式委派 ProfileLoader；新增 `update_scene()` 和 `refresh_scene_cache()` 支持 scene.md 动态维护

### UI 层
- **CliRenderer** — Windows 终端界面（Claude Code 风格）
- **Web UI** — `ui/web_server.py` 仍是旧接口实现，尚未迁移到 v0.5 双消息列表主线

## 三、数据流

### 核心循环（v0.5 双消息列表架构）

```
大脑消息列表                    界说消息列表
┌──────────────────┐           ┌──────────────────┐
│ system(人格+场景+ │           │ system(身份+职责+ │
│       当前状态)   │           │       输出规范)   │
├──────────────────┤           ├──────────────────┤
│ user: 世界消息    │           │ user: 大脑内心独白 │
│ (带[当前状态]头部)│  ← 拼接   │ (干净文本)        │
│                   │           │                   │
│ assistant: 独白   │  ──→     │                   │
└──────────────────┘           │ assistant: 界说输出│
                               │ tool_call/tool_res │
                               └──────────────────┘
```

**每轮循环：**

```
brain.internal_think()
  ↓ 大脑内心独白 → 写入界说 user
narrator.observe()
  ↓ 界说输出 + 可选 tool_call
main.py 拼接 [当前状态] 头部
  ↓ 世界消息 → 写入大脑 user
brain 下一轮感知到世界变化
```

### 无用户时的互相唤醒循环

```
brain.internal_think()
  → 大脑独白写入界说 user
  → narrator.observe()
      → 输出世界消息 + 可选 update_scene()/update_drives() tool_call
      → 拼接 [当前状态] 头部 → 写入大脑 user
  → 10 秒后下一轮
```

### 用户交互流程

```
用户输入
  → narrator.handle_user_arrival/handle_user_message() → 界说旁白
  → brain.respond() → 大脑回复
  → 停止自动观测，等待用户下一次输入
  → 用户超时离开后恢复自主循环
```

## 四、Profile 配置说明

所有人格/世界/场景/记忆从 `profiles/<name>/` 下的 .md 文件读取。

```
python main.py                        # 加载 profiles/default/
python main.py --preset library_den   # 加载 profiles/library_den/
python main.py --preset bedroom_warm --debug  # 加载 + 显示界说原始输出
```

详细规范见 [14-Profile配置系统/02-Profile文件规范.md]。
