## 二、大脑 L2 改进：语义化记忆注入

### 当前问题

L2 日志虽然被读取为 `memory_summary`，但只是简单的文本拼接，没有经过筛选和处理。

### 改进方案

**阶段一（v0.2）：重要性筛选**

```python
def _build_memory_summary(self, max_entries=5):
    logs = self.memory.get_recent_logs(20)  # 读更多再做筛选
    
    # 按权重排序
    weighted = sorted(logs, key=lambda x: x.get("weight", 1), reverse=True)
    
    # 去重（连续相同 type 的内容合并）
    deduped = []
    for log in weighted:
        if not deduped or log["type"] != deduped[-1]["type"]:
            deduped.append(log)
    
    # 截取最重要的事件
    return self._format_events(deduped[:max_entries])
```

**阶段二（v0.3）：LLM 辅助摘要**

当 L2 日志中的事件数量超过阈值时，用一次额外的 LLM 调用生成摘要：

```python
if len(logs) > 20:
    summary = self._summarize_logs(logs)  # LLM 调用
    return f"【今日回顾】{summary}"
```

## 三、大脑 L3：核心记忆系统

### 设计目标

L3 是跨会话的长期记忆，类比人类的"人格记忆"——不是事无巨细地记住每一餐吃了什么，而是记住"我喜欢烹饪"和"我讨厌等待"这种经过提炼的自我认知。

### 存储方案

推荐使用独立的 JSON 文件（非 JSONL），因为 L3 的数据量小、结构稳定：

```python
# memory/brain_l3.json
{
    "version": 1,
    "last_updated": "2026-05-04T00:00:00",
    "personality_evolution": {
        "core_traits": {
            "gentle": 95,      # 可以微调
            "curious": 72      # 根据经历增长/衰减
        },
        "recent_shifts": ["今天和用户聊了烹饪，对食物的兴趣增加了"]
    },
    "core_memories": [
        {
            "summary": "喜欢在午后泡茶，茉莉花茶和云片糕是常备的",
            "importance": 0.8,
            "tags": ["食物", "习惯"],
            "created": "2026-05-03"
        }
    ],
    "experience_summary": [
        "今天在厨房做了一顿饭，感觉很好",
        "和用户进行了愉快的对话"
    ]
}
```

### 注入时机

L3 内容在以下时机注入 LLM 上下文：
1. **每日首次启动**——注入完整的 L3 摘要
2. **每轮思考**——只注入最近的一条 core_memory（控制在 50 token 以内）
3. **触发特定标签时**——例如大脑想到"食物"时，注入带有 "食物" 标签的核心记忆

## 四、界说 S2：世界设定系统

### 当前状态

S2 由 [world/settings.json](world/settings.json) 承载，WorldBuilder.__init__() 时一次性加载到内存，运行期只读。

### 改进方案

```python
class WorldSetting:
    """S2 世界设定管理系统"""
    
    def __init__(self):
        self.settings_file = "world/settings.json"
        self._settings = self._load()
        self._runtime_overrides = {}  # 运行期修改的设定
    
    def get(self, key):
        """读取设定（运行时覆盖优先）"""
        return self._runtime_overrides.get(key, self._settings.get(key))
    
    def set(self, key, value):
        """运行期修改设定"""
        self._runtime_overrides[key] = value
        self._log_change(key, value)
    
    def persist(self):
        """将运行期修改写入文件（可选）"""
        merged = {**self._settings, **self._runtime_overrides}
        with open(self.settings_file, "w") as f:
            json.dump(merged, f, ensure_ascii=False, indent=2)
```

```python
def get_relevant_events(self, scene_name=None, event_types=None, n=5):
    """按条件筛选 S3 事件"""
    events = self._read_all()
    if scene_name:
        events = [e for e in events if scene_name in e.get("description", "")]
    if event_types:
        events = [e for e in events if e.get("type") in event_types]
    return events[-n:]
```

## 六、跨会话持久化

### 启动恢复流程

```
系统启动
    │
    ├──▶ 读取 L3（如果存在）
    │       └── 注入大脑 system prompt 的核心记忆
    │
    ├──▶ 读取 S3 昨日最后 5 条事件
    │       └── 恢复世界状态
    │
    ├──▶ S1 从 S3 重建
    │       ├── 找到最近一次 scene_created
    │       └── 设置当前场景
    │
    └──▶ 正常启动流程
```

### 数据保留策略

| 数据类型 | 保留期限 | 清理策略 |
|----------|----------|----------|
| L2 JSONL | 7 天 | 自动删除 7 天前的日志 |
| S3 JSONL | 7 天 | 同上 |
| L3 JSON | 永久 | 手动管理 |
| S2 JSON | 永久 | 手动修改 |
