## 一、当前时间系统

### 1.1 三层时间感知

当前系统有三个层面的时间感知：

| 层面 | 实现方式 | 精度 | 用途 |
|------|----------|------|------|
| 真实时间 | `datetime.now()` | 毫秒 | 日志时间戳、世界 Builder 获取时段 |
| 思考间隔 | `time.time()` + THINK_INTERVAL | 10秒 | 控制大脑思考频率 |
| 超时检测 | `time.time()` + USER_TIMEOUT | 600秒 | 检测用户是否离开 |

### 1.2 时间在上下文中的呈现

大脑每轮收到的 user 消息中包含当前时间：

```
当前时间：2026-05-03 15:30
```

界说构建场景时包含时段描述：

```python
"time": self.world.get_time_description()
# 例如："午后，光线明亮而温暖。"
```

时段判定基于真实世界的当前时间：

```python
def get_time_description(self):
    hour = datetime.now().hour
    if 5 <= hour < 12:     return "早晨，阳光柔和地洒进来。"
    elif 12 <= hour < 17:  return "午后，光线明亮而温暖。"
    elif 17 <= hour < 21:  return "傍晚，天空染上橘红色。"
    else:                  return "深夜，一切都静悄悄的。"
```

### 1.3 界说端的"时间推进"机制

当大脑连续安静思考时，界说会主动推进时间：

| quiet_rounds | 行为 | LLM 指令 |
|-------------|------|----------|
| 1-2 | 提示安静，但不强制 | "这是大脑连续第N轮安静思考" |
| 3+ | 强制时间流逝 | "让时间自然流逝——描述光线变化、窗外声音、时间推移等（25字以内）" |

## 二、昼夜循环设计（v0.3 目标）

### 2.1 四阶段循环

```
🌅 清晨 (05:00-08:00) — 唤醒阶段
    界说：构造起床场景
    → "新的一天开始了，你睁开眼睛，阳光洒进房间……"
    → 大脑认知重置：新的一天
    → 驱动力重置：fatigue 从低位开始

☀️ 白天 (08:00-18:00) — 活跃阶段
    正常的大脑-界说互相唤醒循环
    用户可随时到达/离开

🌆 傍晚 (18:00-21:00) — 过渡阶段
    光线变暗，氛围转变
    界说的旁白自然偏向安静

🌙 深夜 (21:00-05:00) — 维护阶段
    界说：通知大脑该做日终总结
    大脑：L2→L3 记忆压缩
    界说：构造入睡场景
    → 可能进入"睡眠模式"
```

### 2.2 唤醒与睡眠

**睡眠条件：**
- 深夜时段（hour >= 21 或 hour < 5）
- 连续 10+ 轮安静
- 用户不在场景中

**唤醒条件：**
- 早晨时段（hour >= 5）
- 用户到达

### 2.3 睡眠模式

进入睡眠后的大脑行为变化：

```python
def should_enter_sleep(self, hour, quiet_rounds, user_present):
    """判断是否应该进入睡眠模式"""
    is_night = hour >= 21 or hour < 5
    return is_night and quiet_rounds >= 10 and not user_present

def internal_think_sleep(self):
    """睡眠模式下的思考（精简版）"""
    sleep_thoughts = [
        "……（睡得很沉）",
        "……（翻了个身）",
        "……（梦见了什么，但记不清了）"
    ]
    # 降低 LLM 调用频率（每 60 秒一次）
    # 或者直接返回预设文本，不调用 LLM
    return random.choice(sleep_thoughts)
```

## 三、时间加速系统

### 3.1 问题

在长时间的用户静默期（例如用户睡觉时），真实时间的流逝速度对于虚拟生命来说太慢了。大脑以 10 秒间隔思考，即使运行 8 小时，也才 2880 轮，不足以模拟"一天过去了"。

### 3.2 时间加速机制

```python
class TimeAccelerator:
    """时间加速器——在没有用户时加速时间流逝"""
    
    def __init__(self):
        self.real_start = time.time()
        self.virtual_time = datetime.now()
        self.acceleration = 1.0  # 1x 正常速度
    
    def update(self, user_present):
        """更新虚拟时间"""
        real_elapsed = time.time() - self.real_start
        
        if not user_present:
            # 无用户时：加速时间流逝
            # 前 5 分钟 1x，之后逐渐加速到 60x
            if real_elapsed > 300:  # 5 分钟后
                self.acceleration = min(60, 1 + real_elapsed / 10)
            else:
                self.acceleration = 1
        else:
            # 有用户时：真实时间
            self.acceleration = 1.0
            self.real_start = time.time()  # 重置基准
        
        # 更新虚拟时间
        virtual_elapsed = real_elapsed * self.acceleration
        self.virtual_time = datetime.now() + timedelta(seconds=virtual_elapsed)
    
    def get_time_context(self):
        """获取时间上下文（替代 datetime.now()）"""
        hour = self.virtual_time.hour
        period = self.get_period(hour)
        return {
            "datetime": self.virtual_time,
            "period": period,
            "acceleration": self.acceleration
        }
```

### 4.1 场景的时间敏感度

不同场景在不同时段应该有不同表现：

class TimeSensitiveScene:
    """时间敏感场景——场景状态随时间变化"""
    
    def __init__(self):
        self.scene_states = {
            "卧室": {
                "morning": {"light": "柔和晨光", "sound": "窗外鸟鸣", "activity": "起床"},
                "afternoon": {"light": "明亮温暖", "sound": "远处人声", "activity": "休闲"},
                "evening": {"light": "橘红余晖", "sound": "归巢鸟叫", "activity": "放松"},
                "night": {"light": "漆黑一片", "sound": "万籁俱寂", "activity": "睡眠"}
            },
            "厨房": {
                "morning": {"light": "清新明亮", "sound": "煮水声", "activity": "准备早餐"},
                "afternoon": {"light": "阳光满屋", "sound": "时钟滴答", "activity": "下午茶"},
                "evening": {"light": "暖色灯光", "sound": "炉火声", "activity": "准备晚餐"},
                "night": {"light": "黑暗", "sound": "冰箱嗡鸣", "activity": "沉寂"}
            }

    def get_scene_state(self, scene_name, hour):

## 五、驱动力与时间的关联

### 5.1 时间触发的驱动力变化

```python
def time_triggered_drives(drives, hour):
    """根据时间触发的驱动力调整"""
    if hour == 7:   # 早上 7 点——该吃早餐了
        drives["hunger"] = max(drives["hunger"], 40)
    if hour == 12:  # 中午 12 点
        drives["hunger"] = max(drives["hunger"], 50)
    if hour == 18:  # 晚上 6 点
        drives["hunger"] = max(drives["hunger"], 40)
    if hour >= 22:  # 晚上 10 点后
        drives["fatigue"] = max(drives["fatigue"], 60)
    if hour < 6:    # 凌晨
        drives["fatigue"] = max(drives["fatigue"], 85)
        drives["curiosity"] = min(drives["curiosity"], 20)
    return drives
```

### 5.2 睡眠后的驱动力重置

```python
def on_wake_up(drives):
    """睡醒后的驱动力重置"""
    drives["fatigue"] = 10   # 精力恢复
    drives["hunger"] = 60    # 睡饿了
    drives["curiosity"] = 80 # 新的一天充满好奇
    return drives
```
