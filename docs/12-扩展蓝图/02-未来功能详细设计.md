## 一、技术基础设施

### 1.1 日志与监控系统

**目标：** 建立完善的运行时监控，了解 LLM token 消耗、响应时间、驱动力变化趋势。

```python
class MonitoringSystem:
    """运行时监控"""
    
    def __init__(self):
        self.metrics = {
            "llm_calls": 0,
            "total_prompt_tokens": 0,
            "total_completion_tokens": 0,
            "brain_think_count": 0,
            "narrator_observe_count": 0,
            "scene_changes": 0,
            "user_messages": 0,
            "errors": 0
        }
        self.response_times = []  # [timestamp, duration]
        self.drive_history = []   # 驱动力变化记录
    
    def record_llm_call(self, prompt_tokens, completion_tokens, duration):
        self.metrics["llm_calls"] += 1
        self.metrics["total_prompt_tokens"] += prompt_tokens
        self.metrics["total_completion_tokens"] += completion_tokens
        self.response_times.append({"time": time.time(), "duration": duration})
    
    def get_report(self):
        """生成监控报告"""
        avg_speed = sum(r["duration"] for r in self.response_times[-100:]) / max(len(self.response_times[-100:]), 1)
        return {
            **self.metrics,
            "avg_response_time_100": round(avg_speed, 2),
            "estimated_cost": self.metrics["total_completion_tokens"] * 0.000002
        }
```

### 1.2 配置管理系统

**目标：** 将硬编码的参数抽取到配置文件中，支持运行期热加载。

```python
class ConfigManager:
    """配置管理"""
    
    def __init__(self, config_file="config.yaml"):
        self.config_file = config_file
        self.config = self._load()
        self.watchers = {}
    
    def get(self, key, default=None):
        """支持点号路径：config.get("memory.token_budget")"""
        keys = key.split(".")
        value = self.config
        for k in keys:
            if isinstance(value, dict):
                value = value.get(k)
            else:
                return default
        return value if value is not None else default
    
    def watch(self, key, callback):
        """注册配置变更监听"""
        self.watchers[key] = callback
```

配置示例（config.yaml）：

## 四、性能优化

### 4.1 LLM 调用优化

| 策略 | 预期效果 | 实现难度 |
|------|----------|----------|
| 流式输出 | 首 token 延迟降低 30% | 低 |
| 请求缓存 | 重复场景描述不重复调用 | 中 |
| 批量查询 | 非关键路径并行调用 | 中 |
| 本地小模型 | 零延迟 | 高 |
| Prompt 压缩 | 减少输入 token 数 | 低 |

### 4.2 内存优化

| 策略 | 预期效果 |
|------|----------|
| messages 列表上限 | 最多保留 50 轮 |
| conversation_log 上限 | 最多 200 条 |
| L2 日志分页读取 | 避免大文件全量加载 |
| 旧的 JSONL 自动归档 | 按日分割，7 天后清理 |
