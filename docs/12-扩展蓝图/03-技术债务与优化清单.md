## 二、代码质量问题

### 2.1 代码异味

| 问题 | 描述 | 建议 |
|------|------|------|
| 硬编码配置 | THINK_INTERVAL 等常量在代码中 | 抽取到 settings.json 或 config.yaml |
| 重复的 token 裁剪 | Brain/Narrator 各有相同的 `_trim_messages` | 抽取为共享工具函数 |
| 重复的 LLM 调用 try/catch | 每个方法都重复 try/except | 抽取为装饰器或共享方法 |
| 神秘的 `del messages[1:3]` | 魔术数字，可读性差 | 改为具名方法 `_drop_oldest_round()` |
| import sys.path hack | brain/ 和 narrator/ 中的 `sys.path.insert` | 改为正确的包结构 |
| `__main__._brain` 全局引用 | CLI 渲染器通过全局引用获取状态 | 改为依赖注入 |

### 2.2 测试覆盖

当前测试覆盖率为 **0%**。建议从以下角度开始：

```python
# tests/test_token_estimation.py

    assert _estimate_tokens("你好世界") == 4  # 8 chars // 2

# tests/test_context_builder.py
def test_brain_context_structure():
    builder = BrainContextBuilder(mock_memory)
    messages = builder.build_think_context()
    assert messages[0]["role"] == "system"
    assert "当前时间" in messages[-1]["content"]

# tests/test_narrator_parser.py
def test_scene_change_parsing():
    narrator = NarratorAgent(mock_llm)
    result = narrator._handle_new_scene("[场景:厨房] 你走进厨房。")
    assert result["action"] == "scene_change"
    assert result["scene"]["name"] == "厨房"
```
