## 二、代码质量问题

### 2.1 代码异味

| 问题 | 描述 | 建议 |
|------|------|------|
| 硬编码配置 | THINK_INTERVAL 等常量在代码中 | 抽取到 settings.json 或 config.yaml |
| 重复的 token 裁剪 | Brain/Narrator 各有相同的 `_trim_messages` | 抽取为共享工具函数 |
| 重复的 LLM 调用 try/catch | 每个方法都重复 try/except | 抽取为装饰器或共享方法 |
| 神秘的 `del messages[1:3]` | 魔术数字，可读性差 | 改为具名方法 `_drop_oldest_round()` |
| import sys.path hack | brain/ 和 narrator/ 中的 `sys.path.insert` | 改为正确的包结构 |
| `__main__._brain` 全局引用 | CLI 渲染器通过全局引用获取状态 | 改为依赖注入 |

### 2.2 测试覆盖

当前已有 pytest 测试套件，覆盖 token 估算、ContextBuilder、BrainAgent、NarratorAgent、WorldBuilder、Memory、CLI renderer、LLMClient、ScenesoulLoop 和旧 Web API 的轻量分支。详见 [15-测试/01-测试方案.md](../15-测试/01-测试方案.md)。

主要缺口：
- Web UI 尚未迁移，因此缺少 v0.5 Web 集成测试。
- 未启动真实 CLI 主循环做端到端测试。
- 未设置覆盖率门槛。
