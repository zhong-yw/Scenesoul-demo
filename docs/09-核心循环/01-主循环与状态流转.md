# 主循环与状态流转

> **文档标识：** 09-核心循环/01-主循环与状态流转  
> **对应代码：** [main.py](main.py) `ScenesoulLoop` 类  
> **版本：** v0.5（线程化 LLM 调用 + 双消息列表）

## 一、核心循环概述

系统有两种运行模式：CLI 模式和 Web UI 模式。核心逻辑相同——双消息列表驱动的大脑-界说互相唤醒循环 + 用户消息处理子循环——但驱动方式不同。

| 模式 | 循环驱动 | 输入方式 | 输出方式 |
|------|----------|----------|----------|
| CLI | 50ms 轮询，`time.sleep(0.05)` | `ReadConsoleInputW()`（IME 兼容） | 终端 ANSI 渲染（DECSTBM） |
| Web UI | 前端 5 秒轮询 `/api/think` | HTTP POST `/api/send` | 前端渲染 |

## 二、线程化架构

v0.5 的关键改进：LLM 调用在后台线程执行，主线程保持键盘响应。

```
主线程（50ms 轮询）                  后台线程（LLM 调用）
┌──────────────────────────┐      ┌──────────────────────────┐
│ while running:           │      │ _background_task(loop,   │
│   loop._drain_results()  │◄─────│   task)                  │
│   处理键盘输入            │      │   _task_handle_user_input│
│   超时检测               │      │   _task_run_inner_loop   │
│   定时思考               │      │   _task_handle_user_timeout│
│   time.sleep(0.05)       │      └──────────────────────────┘
└──────────────────────────┘
         │                          ▲
         │ _submit_task()           │ _result_queue.put()
         └──────────────────────────┘
```

### 核心约束

- **单后台任务**：同一时刻只有一个 LLM 任务运行（避免消息列表竞态）
- **用户输入排队**：LLM 忙时消息排入 `_task_queue`，完成后自动处理
- **线程安全渲染**：`RLock` 保护 stdout，后台线程通过 `show_thinking()` 输出状态

## 三、双消息列表架构

v0.5 最核心的变化：大脑和界说各自拥有独立的 message 列表。

```
大脑消息列表 (brain_messages)       界说消息列表 (narrator_messages)
┌──────────────────────────────┐   ┌──────────────────────────────┐
│ system: 【你的人格】          │   │ system: 【你的身份】         │
│         【行为规范】          │   │         【你的职责】         │
│         【世界的场景】        │   │         【输出规范】         │
│         【当前状态】          │   │         【场景维护规则】     │
├──────────────────────────────┤   ├──────────────────────────────┤
│ user: [当前状态] 时间:…      │   │ user: 大脑内心独白（干净文本）│
│       [界说输出的世界消息]    │   │                              │
│                              │   │ assistant: 界说输出          │
│ assistant: 大脑独白          │   │ tool_call: update_scene()    │
│                              │   │ tool_result: success         │
│ user: [当前状态] 时间:…      │   │                              │
│       [场景:厨房] 你走进…    │   │ user: 下一轮大脑独白         │
│                              │   │                              │
│ assistant: 到厨房了……       │   │ ...                          │
└──────────────────────────────┘   └──────────────────────────────┘
```

### 自主循环数据流

```
后台任务 _task_run_inner_loop：
  ① tick_drives()
  
  ② brain.internal_think(brain_messages, drives, scene_info)
     → 返回大脑内心独白文本
  
  ③ 大脑独白写入界说消息列表
     narrator_messages.append({"role": "user", "content": thought})
  
  ④ narrator.observe(narrator_messages)
     → 返回界说输出（narration）+ 可选 tool_call
  
  ⑤ 界说输出追加到界说消息列表
     narrator_messages.append({"role": "assistant", "content": narration})
  
  ⑥ 如果有 tool_call：
     执行 world.update_scene() → 刷新场景缓存
     tool_result 追加到界说消息列表
  
  ⑦ 拼接 [当前状态] 头部 + 界说输出
     → 追加到大脑消息列表
```

## 四、用户交互流程

### 4.1 用户消息处理（后台任务）

```
后台任务 _task_handle_user_input：
  ① 界说编织用户消息（情境融合模式）
     → handle_user_arrival() 或 handle_user_message()
     → 返回 (narration, scene_name, tool_call)
  
  ② 如果有 tool_call：
     执行 world.update_scene() → 写入 scene.md
  
  ③ 旁白注入两个消息列表
     narrator_inject_narration(narration)
  
  ④ 大脑回应
     brain_respond(user_input)
  
  ⑤ 停止，等待用户下次输入（不调用 narrator_observe）
```

### 4.2 用户介入循环

```
用户发消息 → 界说编织 → 大脑回答 → 停，等输入
用户再发   → 界说编织 → 大脑回答 → 停，等输入
...
超时无输入 → 界说构建离开旁白 → 恢复自主循环
```

### 4.3 场景操作分类

| 操作 | 判断条件 | `current_scene_name` 更新 | brain_messages 注入 |
|------|---------|--------------------------|-------------------|
| 增加场景 | `wr["is_new"] == True` | ✅ | 旁白 + `【场景名】描述` |
| 修改场景 | `wr["is_new"] == False` + tool_call | ✅ | 仅旁白 |
| 切换场景 | 无 tool_call，`result["scene_name"]` 存在 | ✅ | 仅旁白 |

## 五、CLI 主循环

```python
POLL_INTERVAL = 0.05
while renderer.running:
    # ① 排空后台任务结果
    loop._drain_results()

    # ② 键盘输入（ReadConsoleInputW，IME 兼容）
    if _kbhit and _getwch:
        while _kbhit():
            ch = _getwch()
            if not ch:
                continue
            renderer.handle_key(ch)
            if renderer.input_submitted:
                user_text = renderer.get_submitted_text()
                if user_text.startswith("/"):
                    renderer.execute_command(user_text)
                elif user_text:
                    loop.handle_user_input(user_text)

    now = time.time()

    # ③ 用户超时检测（600 秒无输入）
    if not loop.processing and loop.user_present and now - loop.last_user_time > USER_TIMEOUT:
        loop.handle_user_timeout()

    # ④ 定时内心独白（10 秒间隔）
    if not loop.processing and not loop.user_present and now - loop.last_think_time >= THINK_INTERVAL:
        # 夜间高疲劳 → 睡眠模式
        # 否则 → run_inner_loop()
        loop.run_inner_loop()

    time.sleep(POLL_INTERVAL)
```

## 六、状态变量总表

| 变量 | 类型 | 位置 | 意义 | 初始值 |
|------|------|------|------|--------|
| `brain_messages` | list | ScenesoulLoop | 大脑消息列表 | [] |
| `narrator_messages` | list | ScenesoulLoop | 界说消息列表 | [] |
| `drives` | dict | ScenesoulLoop | 驱动力 {hunger, fatigue, curiosity} | {30,10,60} |
| `current_scene_name` | str | ScenesoulLoop | 当前场景名 | 从 scene.md 读取 |
| `user_present` | bool | ScenesoulLoop | 用户是否在场景中 | False |
| `sleep_mode` | bool | ScenesoulLoop | 是否处于睡眠状态 | False |
| `last_think_time` | float | ScenesoulLoop | 上次大脑思考的时间戳 | time.time() |
| `last_user_time` | float | ScenesoulLoop | 上次用户消息的时间戳 | 0 |
| `processing` | bool | ScenesoulLoop | 后台任务是否运行中 | False |
| `_task_queue` | Queue | ScenesoulLoop | 排队的用户输入 | empty |
| `_result_queue` | Queue | ScenesoulLoop | 后台任务结果 | empty |
| `narrator.quiet_rounds` | int | NarratorAgent | 连续安静观测轮次 | 0 |
| `brain.last_thought` | str | BrainAgent | 上次大脑独白/回应 | "" |
