# Message 系统设计方案 (v0.5)

> 定稿时间：2026-05-05 18:47
> 状态：已实现（2026-05-05 19:15）

---

## 一、核心思想

大脑与界说各自拥有**独立的 message 列表**，互不共享。

```
┌─────────────────────────────────────┐
│         大脑消息列表                 │
│  system(人格/行为/世界/场景/当前状态)│
│  ─────────────────────────────────  │
│  user: [当前状态] 时间:…            │  ← main.py 拼接
│  [界说输出的世界消息]                │     界说原始输出 + 当前状态头部
│                                     │
│  assistant: 嗯……刚醒。              │
│  ─────────────────────────────────  │
│  user: 阳光透过窗帘洒进来……         │
│  assistant: 暖暖的……有点不想起来。  │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│         界说消息列表                 │
│  system(身份/职责/输出规范/世界规则) │
│  ─────────────────────────────────  │
│  user: (空——系统触发初始场景)        │  ← 大脑的内心独白
│                                     │     干净的一条文本
│  assistant: [初始场景] 阳光……       │  ← 界说原始输出
│                                     │     不包含 [当前状态] 头部
│  ─────────────────────────────────  │
│  user: 嗯……刚醒。窗外好像有鸟叫。   │  ← 大脑内心独白直接写入
│  assistant: 阳光透过窗帘洒进来……    │
│  ─────────────────────────────────  │
│  user: 阳光洒在身上暖暖的……         │
│  assistant: [场景:厨房] 这是一间……  │
│  tool_call: update_scene("厨房",… ) │
│  tool_result: success               │
└─────────────────────────────────────┘
```

**核心数据流：**

1. 大脑 `internal_think()` → 内心独白
2. 大脑内心独白 → 直接写入界说消息列表的 `user` 消息
3. 界说 LLM 调用 → 得到输出文本 + 可选 tool_call
4. 界说输出文本 → 追加到界说自身消息列表（assistant role）
5. 界说输出文本 + main.py 拼接 `[当前状态]` 头部 → 追加到大脑消息列表（user role）
6. 如有 tool_call → 更新 scene.md → 重写大脑固定 system 的场景列表

## 二、大脑的 Message 结构

### 2.1 固定 system（第 1 条）

启动时加载，Profile 切换时重建，运行期随场景列表增长而整体重写。

```
system: """
  【你的人格】——(从 soul.md 加载)
  （人格特质、核心价值）

  【行为规范】
  - 你以第一人称「我」的口语化方式表达内心独白，像在心里对自己说话
  - 你不描述自己的动作——你的行动由外界叙述
  - 你有驱动力：饿了想吃、累了想歇、无聊了想探索

  【世界的模样】
  （从 world.md 加载——世界规则与氛围基调）

  【你的世界感知】
  对话历史中所有以 [当前状态] 开头的消息，都是你感知到的世界本身。
  那是你的现实，不是别人告诉你的事。
  你在其中，它们就是你正在经历的一切。

  【你的日常节奏】
  - 请以第一人称写一段内心独白
  - 如果你感受到驱动力变化（饿了、累了、无聊了），自然会想去做相应的事
  - 用户出现时，自然地回应对方
  - 用户离开后，继续你的日常

  【世界的场景】
  以下是你可到达的所有场景，每个场景都有它当下的样貌：
  （场景列表由界说通过 scene.md 维护，新增场景时整体重写此处）

  【当前状态】
  时间：{当前时间}
  场景：{当前场景名}
  驱动力：{驱动力数值}
"""
```

> **当前状态**有两处：固定 system 中有一份（作为静态基准），每轮 user 消息头部还会带一份实时版本（见 2.3）。前者是当前轮的基准锚点，后者是每轮的最新快照。

### 2.2 对话历史区（messages[1..n]）

#### 无用户时：自我演进

```
[当前状态] 时间:2026-05-05 18:47 场景:卧室 驱动力:hunger:50,fatigue:15,curiosity:20
[初始场景] 一间温馨的卧室，清晨的阳光透过窗帘的缝隙洒进来……

嗯……刚醒。窗外好像有鸟叫。

阳光透过窗帘洒进来，暖暖的。

阳光洒在身上暖暖的……有点不想起来。不过好像有点饿了。

[当前状态] 时间:2026-05-05 18:52 场景:厨房 驱动力:hunger:65,fatigue:24,curiosity:14
[场景:厨房] 这是一间朝西的小厨房，窗台上摆着几盆薄荷，灶台是白色瓷砖的。你推开卧室的门走了进来。

到厨房了。薄荷味……看看有什么吃的。
```

设计要点：
- **user** = 界说输出的世界消息，**没有额外符号标记**
- 每轮 user 消息头部携带 `[当前状态]`（由 main.py 拼接）
- **assistant** = 大脑内心独白（或对用户回复）
- 界说输出进入 message 历史，大脑下一轮能感知到世界变化
- 场景切换时固定 system 的场景列表同步重写

#### 用户出现：两种模式

**模式A — 原生模式（分离）**

用户消息和界说旁白分开为两条独立 user 消息。

```
一个人影出现在厨房门口，光线勾勒出他的轮廓。他探了探头，好像在找人。  ← 界说先构建到达
喂？有人在吗？                                                                ← 用户原始消息

嗯？——我在厨房呢。你来得正好，我正想煮碗面，饿坏了。你也来一碗？
```

```
你看了看灶台，又看了看冰箱，似乎在等对方先开口。
我不饿，你找你的

好，那我看看冰箱里有什么……哎有鸡蛋和番茄，煮碗面吧。
```

**模式B — 情境模式（融合 / 推荐）**

由界说将用户消息直接编织融入当前场景，一条 user 消息完成。

```
厨房门口传来一个声音："喂？有人在吗？"你回过头，看见一个人影站在门口，光线勾勒出他的轮廓。

嗯？——我在厨房呢。你来得正好，我正想煮碗面，饿坏了。你也来一碗？
```

```
你看了看他，他摆摆手说："我不饿，你找你的。"目光落回灶台上的锅。

好，那我看看冰箱里有什么……鸡蛋番茄面，简单又暖和。
```

#### 用户离开时

```
对方安静地转身离开了，脚步声渐渐远去。厨房里只剩下灶台上咕嘟咕嘟煮着的水。

走啦……也好，一个人安静吃碗面。
```

#### 消息类型识别规则

| 来源 | 特征 | 大脑视作 |
|------|------|---------|
| 界说输出 | 带 `[当前状态]` 头部 | 世界消息——正在经历的现实 |
| 用户消息 | 无 `[当前状态]` 头部 | 对方说的话 |

**没有特殊符号前缀**，靠头部字段自然区分。

## 三、界说的 Message 结构

### 3.1 固定 system（第 1 条）

```
system: """
  【你的身份】
  你是白界的「界说」——世界的创造者、叙述者、观测者。
  你没有自我意识，你只负责构造和叙述这个世界。
  你的输出将被大脑感知为他身处的现实。

  【你的职责】
  你收到的每条 user 消息都是大脑的内心独白——那是意识流，是思绪，是感受。
  你需要根据大脑的思绪，决定如何推进这个世界：

  1. 大脑想去新地方 → 构造场景切换
     - 输出格式：[场景:场景名] 场景描述 + 旁白
     - 必须同步调用 update_scene() 写入 scene.md

  2. 大脑在当前场景中有行动意图 → 推进剧情
     - 输出格式：[推进] 叙述发生了什么

  3. 大脑只是安静感受 → 输出氛围旁白
     - 直接输出一小段氛围描写（不超过20字）
     - 如果连续多轮安静，让时间自然流逝

  4. 大脑输出空内容 → 保持安静，本轮不输出

  【输出规范】
  - 使用第二人称「你」描述大脑的行为
  - 新场景的描述要完整、有细节
  - 你的原始输出不包含 [当前状态] 头部——那是外部系统负责拼接的
  - 你的原始输出本身就是进入大脑消息列表的正文部分

  【场景维护规则】
  - 创建新场景时：输出世界消息 + 调用 update_scene() 写入 scene.md
  - 已有场景非关键变化：仅输出世界消息
  - 关键持久状态变化：输出世界消息 + update_scene() 更新 scene.md

  【世界的模样】
  （从 world.md 加载——世界规则、物理规则、氛围基调）
"""
```

### 3.2 对话历史区（messages[1..n]）

```
刚醒来                                                            ← 大脑内心独白直接写入 user
[初始场景] 清晨的阳光透过窗帘的缝隙洒进来……

嗯……刚醒。窗外好像有鸟叫。                                         ← 大脑独白
阳光透过窗帘洒进来，暖暖的。                                         ← 界说输出

阳光洒在身上暖暖的……有点不想起来。不过好像有点饿了。                  ← 大脑独白
[场景:厨房] 这是一间朝西的小厨房，窗台上摆着几盆薄荷……你推开卧室的门走了进来。
[tool_call: update_scene("厨房", "一间朝西的小厨房……")]
[tool_result: success]

到厨房了。薄荷味……看看有什么能吃的。                                 ← 大脑独白
[推进] 灶台上的水壶咕嘟咕嘟冒起了热气。                              ← 界说输出
```

设计要点：
- **user** = 大脑内心独白，**干净的一条文本，没有任务描述、没有驱动力、没有当前场景**
- **assistant** = 界说的输出文本（原始格式，不含 `[当前状态]` 头部）
- **tool_call + tool_result** = LLM 工具调用消息（在界说消息列表中保持完整因果链）
- 界说没有自己独立的 token 裁剪策略——与大腦共享同一套参数

### 3.3 工具调用

界说的 LLM 通过 tool_call 机制调用 `update_scene()`：

```
update_scene(scene_name="书房", description="由老榆木书架围成的小书屋……")
  → 如果 scene_name 已存在：更新描述
  → 如果不存在：追加到 scene.md
```

工具调用流程：
1. 界说 LLM 输出文本 + tool_call（同时返回）
2. tool_call → 由 main.py 执行
3. 执行结果 → `tool_result` 追加到界说消息列表
4. scene.md 更新后 → 整体重写大脑的固定 system（场景列表 + 当前状态）

## 四、main.py 驱动流程

```
轮询循环:
  1. brain.internal_think()           — 大脑内心独白
     → 得到大脑独白文本
  
  2. 如果大脑独白非空:
     a) 大脑独白写入界说消息列表 user
     b) 界说 LLM 调用（system + 多轮历史 + 本轮user）
        → 得到 assistant 文本 + 可选 update_scene/update_drives tool_call
     c) 界说 assistant 追加到界说消息列表
     d) 如果 tool_call: 执行 → tool_result 追加到界说消息列表
     e) 处理 tool_call: 更新 scene.md 或 loop.drives
     f) [当前状态] 头部 + 界说 assistant 文本
        → 拼接后追加到大脑消息列表 user
  
  3. 如果大脑独白为空:
     跳过本轮界说（保持安静）
  
  4. 下一个循环
```

### 初始场景触发

启动时第一次调用，大脑内心独白为标记符或空字符串。界说的 system 末尾规则：
> 如果这是你收到的第一条消息，直接输出初始场景的构建。

### 当前状态拼接（方案2）

界说的原始输出不包含 `[当前状态]` 头部。main.py 在将界说输出追加到大脑消息列表之前，在前面动态拼接：

```
界说原始输出:
  [场景:厨房] 这是一间朝西的小厨房……你推开卧室的门走了进来。

main.py 拼接后 → 大脑消息列表 user:
  [当前状态] 时间:2026-05-05 18:47 场景:厨房 驱动力:hunger:68,fatigue:27,curiosity:12
  [场景:厨房] 这是一间朝西的小厨房……你推开卧室的门走了进来。
```

## 五、界说维护 scene.md 的完整流程

### 5.1 三种场景事件的处理

#### 情况一：新场景创建

```
大脑： "我想找个安静的地方看书"
界说看到"想看书" → 厨房不适合，scene.md 里没有书房

界说输出:
  [场景:书房] 你离开厨房，来到了一间由老榆木书架围成的小书屋……

同时 tool_call:
  update_scene("书房", "一间由老榆木书架围成的小书屋，午后光线透过天窗洒下来，空气里飘着旧纸墨的味道。书桌上摊着一本读到一半的诗集。")

系统: 刷新 scene.md 缓存 → 整体重写大脑固定 system（场景列表 + 当前状态）
```

#### 情况二：已有场景状态更新

```
大脑在厨房煮了面

界说输出:
  [推进] 锅里咕嘟咕嘟煮着面，香气飘了出来。

同时 tool_call（可选——如果场景的持久状态变了）:
  update_scene("厨房", "一间朝西的小厨房……灶台的白瓷砖上有几点油渍，锅里还剩着半碗面汤。")
```

#### 情况三：纯氛围旁白

```
界说输出:
  [观察] 窗外隐约传来鸟鸣声。
  
不触发 scene.md 更新。
```

### 5.2 界说更新 scene.md 的规则

| 场景事件 | 写入大脑消息历史 | 更新 scene.md |
|---------|-----------------|--------------|
| [场景:名称] 新场景 | ✅ user 消息 | ✅ 写入新场景 |
| [场景:名称] 已有场景 | ✅ user 消息 | ❌ 不需要 |
| [推进] 场景变化 | ✅ user 消息 | ⚠️ 如果持久状态变了才更新 |
| [观察] 氛围旁白 | ✅ user 消息 | ❌ 不需要 |

## 六、双轨通道总结

| 通道 | 职责 | 媒介 |
|------|------|------|
| 世界消息（user） | 即时叙述（"你刚煮完面"）+ 当前状态 → 因果链 | 大脑消息列表的 user 消息（头部由 main.py 拼接当前状态） |
| scene.md | 持久状态（场景的当前样子）→ 场景快照 | 大脑固定 system 的【世界的场景】字段 |

## 七、无用户时的自我演进

### 7.1 循环机制

```
1. 大脑 internal_think() → assistant: "有点饿了……"

2. 大脑独白写入界说 user
   界说 LLM 调用 → 输出世界消息 + 可选 tool_call
   界说 world 消息传入大脑消息列表 user

3. 大脑下一轮 internal_think() 感知到世界变化
```

### 7.2 用户插入时

```
1. 用户消息（无 [当前状态] 头部）进入大脑消息列表

2. 界说构建到达/接收旁白
   → 世界消息进入大脑消息列表

3. 大脑 respond()（不是 internal_think）
   → assistant 回复用户
```

### 7.3 用户离开时

```
1. 超时检测（10 分钟无交互）

2. 界说输出离开旁白
   → 对方安静离开了……

3. 大脑感知到用户不在
   → 回到自我演进循环
```
