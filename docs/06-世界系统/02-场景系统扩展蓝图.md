## 一、现状分析

当前场景系统的主要短板：

| 问题 | 影响 | 优先级 |
|------|------|--------|
| `build_scene()` 永远返回默认场景 | 界说无法真正"构建新场景" | P0 |
| 场景描述包含了旁白（行动描述） | 大脑接收的场景信息有歧义 | P0 |
| 无场景模板 | 场景一致性差 | P1 |
| 无场景物体/可交互元素 | 世界"空" | P1 |
| 无天气/环境系统 | 世界变化单调 | P2 |

## 二、LLM 动态场景生成

### 接口设计

```python
def build_scene(self, scene_name, brain_context=None):
    """LLM 动态生成场景"""
    
    # 1. 检查是否有模板（见下文）
    if scene_name in self.settings.get("scene_templates", {}):
        return self._build_from_template(scene_name, brain_context)
    
    # 2. LLM 动态生成
    prompt = f"""你正在为一个虚拟生命构造场景。
    
场景名称：{scene_name}
{'+ 大脑当前状态：' + brain_context if brain_context else ''}

请输出以下格式（JSON）：
{{
    "name": "{scene_name}",
    "description": "场景的静态描述（不包含任何角色的行动描述）",
    "ambient": "场景的氛围、声音、光线等",
    "objects": ["场景中存在的物体列表"],
    "interactions": ["可供交互的元素"]
}}
"""
    # 调用 LLM 获取场景
    result = self.llm.chat([{"role": "user", "content": prompt}])
    scene = json.loads(result["content"])
    scene["time"] = self.get_time_description()
    return scene
```

### 模板变量解析引擎

```python
import random
import re

class SceneTemplateEngine:
    def render(self, template_name, variables=None):
        template = self.settings["scene_templates"][template_name]
        
        # 解析 {variable} 占位符
        def resolve_placeholder(match):
            key = match.group(1)
            if key in (variables or {}):
                return variables[key]
            if key in template.get("variables", {}):
                val = template["variables"][key]
                if isinstance(val, list):
                    return random.choice(val)
                return val
            return f"{{{key}}}"
        
        description = re.sub(r'\{(\w+)\}', resolve_placeholder, template["description_base"])
        
        return {
            "name": template_name,
            "description": description,
            "objects": template["default_objects"].copy(),
            "ambient_sounds": template["ambient_sounds"]
        }
```

## 四、场景物件系统

### 可交互元素

当前世界是"空"的——大脑感知到的物体完全依赖 LLM 在场景描述中提及。为了让世界更真实，需要引入物件系统：

```python
class SceneObject:
    def __init__(self, name, description, states=None, interactions=None):
        self.name = name          # "水壶"
        self.description = description  # "一个银色的不锈钢水壶"
        self.states = states or {
            "position": "灶台上",
            "state": "空的"       # 空的/满水的/正在烧
        }
        self.interactions = interactions or ["拿起", "放下", "倒水"]
```

### 物件状态记录

```python
class SceneState:
    """场景状态跟踪"""
    
    def __init__(self, name):
        self.name = name
        self.objects = {}       # name → SceneObject
        self.npcs = []          # 场景中的NPC
        self.players = []       # 场景中的"用户"
    
    def apply_change(self, action, target, new_state):
        """记录物件状态变化"""
        if target in self.objects:
            self.objects[target].states.update(new_state)
```

## 五、天气与环境系统

### 设计思路

时间推进时，环境不应只有"光线变化"，还应该有天气、声音、温度等多维度变化：

```python
class EnvironmentSystem:
    """环境系统：天气、声音、温度"""
    
    def __init__(self):
        self.weather = "晴"
        self.temperature = 22
        self.ambient_sounds = ["远处鸟鸣"]
        self.light_level = "明亮"
    
    def tick(self, time_delta_minutes=10):
        """按时间推进更新环境"""
        # 天气变化（低频）
        if random.random() < 0.01:  # 1% 概率
            self._change_weather()
        
        # 光线变化（跟随时间）
        self._update_light()
        
        # 声音变化
        self._update_sounds()
    
    def get_description(self):
        """获取当前环境的文本描述"""
        parts = []
        parts.append(f"天气{self.weather}")
        parts.append(f"光线{self.light_level}")
        if self.ambient_sounds:
            parts.append(f"耳边传来{','.join(self.ambient_sounds)}")
        return "，".join(parts)
```

## 六、多场景并发

### 当前限制

大脑一次只能在一个场景中。切换场景时旧场景完全丢弃。

### 多场景设计

```python
class WorldManager:
    """世界管理器：管理多个并发场景"""
    
    def __init__(self):
        self.scenes = {}        # name → SceneState
        self.active_scene = None  # 当前激活场景名
        self.scene_links = {}     # 场景之间的连接关系
        # "卧室" → {"门": "走廊"}
        # "走廊" → {"厨房": "厨房", "书房": "书房"}
    
    def add_scene(self, scene_state):
        self.scenes[scene_state.name] = scene_state
    
    def switch_scene(self, target_name):
        """切换场景（保留旧场景状态）"""
        if self.active_scene:
            # 自动保存当前场景状态
            pass
        self.active_scene = target_name
        return self.scenes.get(target_name)
    
    def connect_scenes(self, scene_a, exit_name, scene_b):
        """建立场景连接"""
        if scene_a not in self.scene_links:
            self.scene_links[scene_a] = {}
        self.scene_links[scene_a][exit_name] = scene_b
```
