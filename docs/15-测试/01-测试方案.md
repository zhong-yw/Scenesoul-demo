# 测试方案：白界 · Scenesoul

> **文档标识：** 15-测试/01-测试方案
> **版本：** v0.5（当前测试套件）
> **最后更新：** 2026-05-07
> **工具：** pytest + unittest.mock

---

## 一、运行方式

```bash
pytest
pytest -q
pytest tests/test_narrator_agent.py::TestObserve::test_scene_change
pytest -k "scene_change"
```

测试使用 mock LLM，不需要真实 API Key。共享 fixture 位于 `tests/conftest.py`。

---

## 二、当前测试结构

```
tests/
├── conftest.py                        # mock LLM、Agent、Memory、WorldBuilder fixture
├── test_token_estimation.py           # token 估算函数
├── test_context_builders_brain.py     # BrainContextBuilder system 重写 + 裁剪
├── test_context_builders_narrator.py  # NarratorContextBuilder system 替换 + 裁剪
├── test_brain_agent.py                # BrainAgent internal_think/respond
├── test_narrator_agent.py             # NarratorAgent observe/user/quiet_rounds
├── test_memory_system.py              # BrainMemory/NarratorMemory JSONL
├── test_world_builder.py              # default scene + update_scene
├── test_cli_renderer.py               # InputLine + 基础命令渲染
├── test_llm_client.py                 # LLMClient 初始化、chat、异常传播
├── test_main_loop.py                  # ScenesoulLoop 核心状态流转
└── test_web_server.py                 # 旧版 Web API 的轻量保护测试
```

---

## 三、覆盖重点

| 模块 | 覆盖内容 | 备注 |
|------|----------|------|
| `context_builders.py` | token 估算、system prompt 替换、消息裁剪 | 保证双消息列表上下文稳定 |
| `brain/brain_agent.py` | `internal_think()`、`respond()`、LLM fallback | BrainAgent 不再持有消息历史 |
| `narrator/narrator_agent.py` | `[场景:]`、`[推进]`、普通观察、空输出、异常、用户到达/发言/离开 | `chat_with_tools()` 通过 mock 覆盖 |
| `world/world_builder.py` | profile 模式、默认场景、`update_scene()` 新增/覆盖 | 写入通过 `tmp_path` 隔离 |
| `memory/memory_system.py` | L1/S1 状态、L2/S3 JSONL 写入读取 | 运行时日志目录通过 patch 隔离 |
| `main.py` | `ScenesoulLoop` 的用户输入、超时、睡眠、旁白注入、tool_call 写回 | 不启动真实 CLI |
| `ui/cli_renderer.py` | 输入缓冲、光标移动、命令处理、输出分隔 | 以单元测试为主 |
| `ui/web_server.py` | 未初始化状态、空消息、轮询等待 | Web UI 仍是旧接口，测试只保留最低限度保护 |

---

## 四、已知测试边界

- 当前测试不进行真实 LLM API 调用。
- 当前测试不启动 Windows 控制台输入循环；`ReadConsoleInputW()` 相关行为主要由 `InputLine` 和 `CliRenderer.handle_key()` 间接覆盖。
- `ui/web_server.py` 尚未迁移到 v0.5 双消息列表，相关测试只覆盖不会依赖旧 Agent 实例的路由分支。
- 尚未引入覆盖率门槛；新增核心流程时建议同步添加单元测试或主循环状态测试。
