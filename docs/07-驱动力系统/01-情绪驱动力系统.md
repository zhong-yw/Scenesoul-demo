## 一、概述

驱动力系统是大脑 Agent 的"生理时钟"和"动机引擎"。通过三个基本维度（饥饿、疲劳、好奇）的自动变化，模拟虚拟生命的内在驱动力——即使没有用户输入，大脑也会因为驱动的变化而产生不同的思考内容。

---

## 二、当前实现

### 驱动力维度

| 驱动力 | key | 初始值 | 范围 | 每轮变化 |
|--------|-----|--------|------|----------|
| 饥饿感 | `hunger` | 0 | 0-100 | +1 |
| 疲劳感 | `fatigue` | 0 | 0-100 | +3 |
| 好奇心 | `curiosity` | 30 | 0-100 | -1 |

### 更新方法

```python
def tick_drives(self):
    d = self.memory.l1["drives"]
    d["fatigue"] = min(100, d.get("fatigue", 0) + 3)
    d["curiosity"] = max(0, d.get("curiosity", 30) - 1)
    d["hunger"] = min(100, d.get("hunger", 0) + 1)
    self.memory.update_l1("drives", d)
```

### 调用时机

在 [main.py](main.py) 的主循环中，每次大脑思考前调用：

```python
# 无用户时的思考循环
if not user_present and now - last_think_time >= THINK_INTERVAL:
    brain.tick_drives()                    # ① 更新驱动力
    thought = brain.internal_think(...)    # ② 大脑思考
    ...

# 用户超时后恢复思考
if user_present and now - last_user_time > USER_TIMEOUT:
    ...
    brain.tick_drives()
    thought = brain.internal_think(...)
```

Web UI 模式下，`/api/think` 接口中**没有调用 `tick_drives()`**——这是一个 Bug。

## 四、当前局限

### 4.1 核心问题：驱动力的数值未影响大脑的行为

这是当前系统最大的缺口——驱动力**只作为数字被展示**，但：

1. **大脑的 system prompt 没有驱动力阈值说明**
   - 没有"当 fatigue > 70 时，你应该感到疲惫"
   - 没有"当 hunger > 60 时，你应该想要吃饭"
   - LLM 需要在没有任何提示的情况下"自己意识"到驱动力数值的含义

2. **界说的观测提示中没有驱动力解读**
   - 界说收到了驱动力数值，但没有被要求"根据驱动力判断大脑的需求"

3. **tick_drives() 在 Web UI 模式下未调用**
   - 这是一个实现漏洞

### 4.2 驱动力维度太少

只有三个维度，且都是"递增或递减的线性值"。理想情况下还应该包括：

| 维度 | 变化逻辑 | 意义 |
|------|----------|------|
| 满足感 | 用户互动后增加，随时间衰减 | 社交需求 |
| 焦虑感 | 有未完成任务时增加 | 任务驱动 |
| 兴奋度 | 新场景/新事件时增加 | 好奇心触发 |
| 孤独感 | 长时间无用户互动时增加 | 社交驱动力 |

### 4.3 无事件触发的驱动力变化

当前驱动力只随时间线性变化。但实际的驱动力应该是行为触发的：

- 用户说"我给你做了顿饭" → 饥饿感归零
- 大脑思考了复杂问题 → 疲劳感额外 +5
- 界说切换新场景 → 好奇心翻倍

## 五、扩展设计：驱动力 2.0

### 5.1 阈值驱动的行为提示

```python
class DriveSystem:
    
    THRESHOLDS = {
        "hunger": {
            30: "你有点饿了，想吃点什么。",
            60: "你肚子在咕咕叫，需要找点吃的。",
            85: "你已经很饿了，急需进食。"
        },
        "fatigue": {
            40: "你有点累了，想坐下来歇会儿。",
            70: "你很疲惫了，应该休息。",
            90: "你已经筋疲力尽，需要立刻睡觉。"
        },
        "curiosity": {
            10: "你对周围的事情提不起兴趣。",
            30: "你有点无聊，想找点事情做。",
            60: "你好奇周围有什么新鲜事。"
        }
    }
    
    def get_drive_prompts(self, drives):
        """根据驱动力数值生成行为提示"""
        prompts = []
        for key, value in drives.items():
            thresholds = sorted(self.THRESHOLDS.get(key, {}).items())
            for threshold, prompt in thresholds:
                if value >= threshold:
                    prompts.append(prompt)
                    break
        return prompts
```

### 5.2 行为触发的驱动力变化

```python
class DriveEvent:
    """驱动力事件"""
    
    @staticmethod
    def on_user_interaction(drives):
        """用户互动时：满足感增加，孤独感减少"""
        drives["satisfaction"] = min(100, drives.get("satisfaction", 50) + 15)
        # 已有维度也可受影响
        drives["curiosity"] = min(100, drives.get("curiosity", 30) + 10)
        return drives
    
    @staticmethod
    def on_scene_change(drives):
        """场景切换时：好奇心重置"""
        drives["curiosity"] = 80  # 新场景激发好奇心
        return drives
    
    @staticmethod
    def on_eat(drives):
        """进食后：饥饿感归零"""
        drives["hunger"] = 0
        drives["fatigue"] = min(100, drives.get("fatigue", 0) - 5)  # 吃饱了有点困
        return drives
    
    @staticmethod
    def on_sleep(drives):
        """睡眠后：疲劳感归零"""
        drives["fatigue"] = 0
        drives["hunger"] = min(100, drives.get("hunger", 0) + 20)  # 睡醒后饿了
        return drives
```

### 5.3 更新频率重构

当前 `tick_drives()` 每个思考周期（10 秒）都调一次。但驱动的变化应该是时间比例的，而非轮次比例的：

```python
def tick_drives_time_based(self, elapsed_seconds):
    """基于真实时间更新驱动力"""
    minutes = elapsed_seconds / 60
    d = self.memory.l1["drives"]
    d["hunger"] = min(100, d.get("hunger", 0) + 2 * minutes)      # 每小时 +12
    d["fatigue"] = min(100, d.get("fatigue", 0) + 3 * minutes)    # 每小时 +18
    d["curiosity"] = max(0, d.get("curiosity", 30) - 1 * minutes) # 每小时 -6
    self.memory.update_l1("drives", d)
```

## 七、界说端驱动力联动

界说不仅应该接收到驱动力数值，还应该利用它来做更精准的场景判断：

### 当前

```python
# 界说只把驱动力作为"上下文信息"传递给 LLM
parts.append(f"大脑驱动力：{json.dumps(brain_drives)}")
```

### 改进

```python
# 界说根据驱动力主动推测大脑意图
drive_hints = []
if brain_drives.get("hunger", 0) > 60:
    drive_hints.append("大脑应该饿了，适合构造与食物相关的场景")
if brain_drives.get("fatigue", 0) > 70:
    drive_hints.append("大脑很疲惫，应该引导他休息")

parts.append(f"大脑驱动力：{json.dumps(brain_drives)}")
if drive_hints:
    parts.append(f"驱动暗示：{'，'.join(drive_hints)}")
```
