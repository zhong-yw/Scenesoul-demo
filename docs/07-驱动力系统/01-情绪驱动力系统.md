# 驱动力系统

> **文档标识：** 07-驱动力系统/01-情绪驱动力系统  
> **对应代码：** [main.py](main.py) `ScenesoulLoop` + [narrator/narrator_agent.py](narrator/narrator_agent.py) `update_drives` 工具  
> **版本：** v0.5

---

## 一、概述

驱动力系统是大脑 Agent 的"动机引擎"。通过 soul.md 中定义的特质（traits）驱动大脑的思考和行为。驱动力完全可自定义，不限于固定的 hunger/fatigue/curiosity。

---

## 二、驱动力定义

驱动力从 `profiles/<名称>/soul.md` 的 YAML frontmatter `traits` 字段读取：

```yaml
---
core: 默认陪伴型
traits:
  温柔: {value: 80, desc: "对他人的关怀程度。高时说话体贴，低时冷漠疏离。"}
  好奇: {value: 60}   # desc 可选
  平静: 75            # 简写
---
```

- **特质名完全自定义**：可以是中文或英文，任意名称
- **value**：数值，默认取值范围 -100~100
- **desc**：可选，描述该特质的含义和高低值的表现。为空时按特质名字面意思理解
- **读取时机**：`ScenesoulLoop.__init__()` 调用 `ProfileLoader.get_initial_drives(profile_name)`

## 三、驱动力更新

驱动力只能由界说通过 `update_drives` 工具更新，不自动衰减。

界说通过 `update_drives` 工具根据场景/行为调整驱动力：

```python
UPDATE_DRIVES_TOOL = {
    "type": "function",
    "function": {
        "name": "update_drives",
        "description": "更新大脑的驱动力/特质状态。根据场景、事件、大脑的行为调整数值。",
        "parameters": {
            "type": "object",
            "properties": {
                "drives": {
                    "type": "object",
                    "description": "要更新的驱动力键值对，如 {\"好奇\": 80, \"平静\": 30}。只传需要变化的项。"
                }
            },
            "required": ["drives"]
        }
    }
}
```

界说在 `observe()` 中通过 `chat_with_tools()` 调用此工具。main.py 处理 tool_call 后更新 `self.drives` 并 clamp 到 0-100。

### 3.2 更新规则

界说的 narrator.md 中定义了更新指引：
- 大脑吃了东西 → 降低「饥饿」
- 大脑休息了 → 降低「疲惫」
- 大脑探索新事物 → 升高「好奇」
- 大脑和用户愉快对话 → 升高「活泼」
- 夜晚降临 → 升高「疲惫」，降低「平静」
- 只在有明显变化时更新，不要每轮都调

## 五、睡眠模式

当满足以下条件时，大脑进入睡眠：
- 夜间（hour >= 21 或 < 6）
- 存在疲劳相关驱动力（键名包含"疲"或"fatigue"）且值 >= 80

```python
fatigue_val = 0
for k, v in loop.drives.items():
    if "疲" in k or "fatigue" in k.lower():
        fatigue_val = max(fatigue_val, v)
```

## 六、扩展建议

| 方向 | 说明 |
|------|------|
| 阈值行为提示 | 在 system prompt 中根据驱动力数值注入行为暗示 |
| 事件触发 | 用户说"我给你做了顿饭" → 饥饿归零 |
| 场景联动 | 切换新场景 → 好奇心升高 |
| 时间加速 | 空闲时驱动力变化更快 |
